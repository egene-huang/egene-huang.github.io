记一次hexo默认markdown引擎更换

Posted on 2017-02-21 | In hexo |

使用hexo搭建好博客以来，一直各种折腾，平时记录笔记生成的静态网页的时候总是会不太顺利, 基本上都会话费好长一段时间来排除Markdown标签导致的html代码生成的错误. 好几次都只好把一篇完整的日记拆分为两个. 因为我习惯的那些Markdown标签在hexo默认的Markdown引擎中不能识别，所以不能生成正确的html代码。提供的默认Markdown解析引擎是hexo-renderer-marked.

marked引擎是最基本的markdown解释引擎，对后来的很多markdown扩展表强支持不是很有好，例如: ``` 语法

我经常需要使用这个标签,因为我经常贴系统配置/编程语言部分代码等,后来在知乎上找到一款marked引擎的替代者hexo-renderer-markdown-it

<!-- 添加到站点_config.yml 下面 -->
# hexo # markdown # hexo-renderer-markdown-it
